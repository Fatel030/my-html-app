「社交與職場互動指南」創作心得與技術分享

發佈日期：2025年6月11日 | 作者：蔡宗諭

專案緣起與目標

製作「社交與職場互動指南」互動式網頁應用程式的初衷，源於對現有報告呈現形式的思考。 傳統的 PDF 或靜態網頁報告，雖然內容豐富，但在閱讀體驗和互動性方面仍有提升空間。 我希望透過一個單頁應用程式 (SPA) 的形式，將報告中的核心知識點，以更生動、易於探索和消化的方式呈現給使用者。

我的目標是創造一個直觀、響應式的平台，讓訪客能夠輕鬆點擊、展開資訊，並在互動中加深對社交與職場關鍵能力的理解，將被動閱讀轉化為主動學習。

設計挑戰與解決方案

這個專案最大的挑戰之一是如何將大量文字內容，有效地拆解並重新組織成適合互動的區塊，同時保持資訊的連貫性和邏輯性。

資訊架構設計

我沒有盲目照搬報告的章節結構，而是根據內容的邏輯關係和使用者探索的路徑，設計了「溝通基石」、「建立連結」等核心區塊，並輔以導覽列快速跳轉。這樣，使用者可以線性學習，也可以直接跳到感興趣的主題。

前端技術選型

我選擇了 **HTML** 作為結構基礎，**Tailwind CSS** 進行快速響應式佈局和美觀樣式，以及 **Vanilla JavaScript** 來處理所有互動邏輯（如手風琴效果、分頁切換、導航滾動）。 對於報告中較為量化的數據，雖然不多，但也引入了 **Chart.js** 庫來進行簡單的視覺化呈現，以增強數據的直觀性。

學習與反思

在專案開發過程中，我對前端開發的整個流程有了更深的理解，特別是在以下幾個方面：

  • 資訊設計的重要性： 好的介面設計不僅是美觀，更是資訊的有效傳遞。
  • Tailwind CSS 的高效性： 快速搭建響應式佈局，極大地提升了開發效率。
  • Vanilla JS 的靈活性： 雖然有許多框架，但純 JavaScript 在實現特定互動時依然強大且輕量。
  • 持續學習與迭代： 每次修改和優化，都是一次學習和提升的機會。

未來，我將持續完善這個應用程式，並希望能在其他專案中應用這些寶貴的經驗。
